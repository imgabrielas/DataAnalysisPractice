# Data Cleaning Exercise

A hands-on data cleaning exercise using a synthetic, intentionally messy dataset generated by an LLM.

## Files

| File | Description |
|---|---|
| `set.csv` | Raw dirty data — the starting point |
| `dataCleaning.ipynb` | Jupyter notebook with the full cleaning process, step by step with comments |
| `clean_data.csv` | End product — cleaned and standardized dataset |

## What's wrong with the raw data

The dirty dataset (`set.csv`) contains a variety of common real-world data quality issues:

- Inconsistent casing — names, emails, country names (`USA`, `us`, `United States`)
- Mixed quoting styles (`'`, `"`) causing parsing problems
- Missing values represented inconsistently (empty strings, `NaN`, `None`)
- Inconsistent date formats (`2021-05-12`, `12/06/2020`)
- Mixed state formats — abbreviations vs full names (`CA` vs `California`)
- Age stored as words instead of numbers (`"thirty"` instead of `30`)
- Phone numbers with and without dashes (`555-1234` vs `5551234`)
- Invalid or malformed emails

## Cleaning steps (notebook walkthrough)

1. **Drop null rows** — remove records where any field is missing
2. **Name standardization** — capitalize first/last names consistently
3. **Gender normalization** — uppercase values; add a binary `binary_gender` column (`0`=M, `1`=F)
4. **Email validation** — lowercase all emails; flag invalid ones (missing `@` or domain) and replace with `Unknown`; add `email_valid` boolean column
5. **Phone formatting** — strip dashes for a uniform numeric format
6. **Country normalization** — map variants (`USA`, `us`, `United States`) to `United States of America`
7. **State expansion** — map abbreviations to full names for US states, Canadian provinces, and Mexican states
8. **Age conversion** — convert word-form ages (`"thirty"`) to integers; invalid values fallback to `0`
9. **Loyalty points** — cast to integer type
10. **Date standardization** — parse all date formats to ISO 8601 (`YYYY-MM-DD`) using `errors='coerce'` for invalid entries
